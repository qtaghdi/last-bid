# LAST BID 마일스톤 기록

이 문서는 LAST BID의 Milestone 1부터 Milestone 5까지 완료된 개발 범위와 각 단계에서 확립한 기준을 기록합니다.

역사적 기획 프롬프트를 대신하는 완료 이력이며, 현재 실행 방법과 안정 기준은 `README.md`, 세부 구현 제약은 `AGENTS.md`, 실제 동작은 회귀 테스트를 우선합니다.

## 요약

| 단계 | 목표 | 완료 결과 |
| --- | --- | --- |
| Milestone 1 | 경매 게임의 기본 루프 검증 | 10라운드 경매, 카드 효과, 심판, 승패, 결정론적 RNG |
| Milestone 2 | 정보 비대칭과 NPC 판단 검증 | 참가자별 지식, 조사, 성향별 평가, 대사와 제한적 허세 |
| Milestone 2.5 | 전체 흐름의 UX 와이어프레임 검증 | 단계별 패널, 공통 HUD, 참가자 상태, 결과와 DEBUG UI |
| Milestone 3 | 낙찰 후 위험 관리 확장 | 3단계 봉인, 사고, 보관, 판매, 소각, 소유권 이전 |
| Milestone 4 | 캐릭터 중심 협상 추가 | 협상 단계, 캐릭터 프로필, 감정, Tell, 관계, 비장의 수단 |
| Milestone 5 | 지속되는 약속과 신뢰 결과 추가 | 약속, 이행·위반, 평판, 기억, 결정론적 배신 |

## Milestone 1 — Core Auction Prototype

게임이 한 런을 처음부터 끝까지 진행할 수 있는 최소 플레이 구조를 만들었습니다.

구현 범위:

- 플레이어 1명과 NPC 3명
- 총 10라운드와 단계 기반 게임 흐름
- 오름 입찰, 최소 인상액, 입찰과 패스, 낙찰과 무입찰 처리
- HP, 골드, 생존 상태와 참가자 사망
- 카드 획득, 즉시 효과, 지연 효과와 심판 단계
- 플레이어 사망, 최후 생존자, 10라운드 생존에 따른 승패 판정
- 중앙 Seed RNG와 같은 Seed 결과 재현
- 디버그 로그와 기본 플레이 UI
- 프로토타입 카드 6종

확립한 기준:

- `GameFlowController`가 메인 단계 전환의 유일한 권한을 가집니다.
- 카드 정의, 카드 인스턴스, 효과 실행, 경매 규칙을 분리합니다.
- 게임플레이 난수는 중앙 RNG를 통해서만 소비합니다.

## Milestone 2 — Information Asymmetry

정확한 카드 효과를 모르는 상태에서 제한된 단서와 NPC 행동을 해석하는 핵심 플레이를 추가했습니다.

구현 범위:

- 실제 카드 정보와 공개·숨김 단서의 데이터 분리
- 플레이어와 NPC마다 독립적인 `KnowledgeState`
- 정보 토큰 2개와 PRE_INFO 추가 조사
- 임시 카드명, 역할군, 위험도 범위, 가치 범위, 발동 시점, 대상 유형으로 구성된 공개 정보
- 수집가, 채권자, 도박사의 주관적 카드 평가와 최대 입찰가
- 상황별 NPC 반응 대사
- 도박사의 제한적인 허세 입찰
- 단서 배분, AI 판단, 대사와 허세의 Seed 재현

정보 경계:

- PRE_INFO와 AUCTION은 내부 ID, 실제 카드명, 정확한 수치와 전체 효과를 노출하지 않습니다.
- NPC 판단은 `CardDefinition.effects`가 아닌 자신이 가진 `KnowledgeState`만 사용합니다.
- 실제 정보와 참가자별 지식·평가 값은 DEBUG에서만 확인합니다.

## Milestone 2.5 — UX Prototype

최종 아트 제작 전에 전체 플레이 흐름과 정보 위계를 검증할 수 있는 텍스트 중심 와이어프레임을 만들었습니다.

구현 범위:

- 공통 HUD: 라운드, 단계, Seed, 정보 토큰, DEBUG 토글
- 참가자 패널: HP, 골드, 보유 카드, 생존·패스·현재 차례와 NPC 반응
- PRE_INFO: 공개 출품 정보, 구조화된 단서, 추가 조사
- AUCTION: 시작가, 현재가, 최고 입찰자, 다음 입찰가와 행동 불가 사유
- 현재 입찰자가 없을 때 `입찰 없음`, 최초 버튼에 `첫 입찰 N G` 표시
- POST_AUCTION, JUDGMENT, ROUND_END, RUN_RESULT 단계별 패널
- 같은 Seed와 새 Seed 재시작
- 기본 닫힘 상태의 별도 DEBUG Drawer
- 1280×720과 1920×1080에서 확장되는 Control/Container 레이아웃
- 공용 Theme과 UI 팔레트

이 단계의 UI는 기능과 정보 구조를 검증하는 와이어프레임입니다. 캐릭터 초상화, 카드 일러스트, 최종 폰트, 오디오와 완성형 연출은 포함하지 않았습니다.

## Milestone 3 — Post-Auction Risk Management

낙찰을 즉시 완전 공개와 효과 실행으로 연결하지 않고, 위험을 관리하는 별도 선택 단계로 확장했습니다.

구현 범위:

- 실제 위험도에 따른 3단계 봉인과 단계별 사고 확률
- 중앙 RNG를 이용한 사고 재현
- 부분 개봉과 봉인 상태 보관
- 세 번째 봉인 개봉 시 실제 이름·효과 공개와 `ON_OPEN` 실행
- 봉인 카드 3장 인벤토리 제한
- NPC 대상 1회 판매 제안과 단서·가격 선택
- 판매 수락 시 골드, 소유권, 지연 카운트 이전
- 카드별 소각 비용, 소각 효과와 지연 효과 취소 정책
- 안정적인 카드 instance ID와 소유권 이전 기록
- 결정론적인 NPC의 OPEN, KEEP, BURN 판단
- 낙찰 후 행동이 완료되기 전 JUDGMENT 진행 차단

소유권 정책:

- `FOLLOW_CURRENT_OWNER`
- `STAY_WITH_ORIGINAL_OWNER`
- `CANCEL_ON_TRANSFER`
- `TRIGGER_ON_TRANSFER`

## Milestone 4 — Character Negotiation

PRE_INFO와 AUCTION 사이에 NPC가 먼저 거래 조건을 제안하는 협상 단계를 추가했습니다.

구현 범위:

- `PRE_INFO → NEGOTIATION → AUCTION` 흐름
- 라운드당 최대 2명의 서로 다른 살아 있는 NPC 제안
- 카드 구매, 봉인 유지, 정보 공유, 경매 불참, 카드 보유 제안
- 수락, 거절과 제안당 한 번의 가격 재제안
- 마라, 볼트, 세라의 데이터 기반 캐릭터 프로필과 대사
- 런마다 선택되는 숨겨진 목표와 진행도
- 감정 상태와 신뢰도 100% 미만의 행동 신호(Tell)
- 런 한정 Relationship `-2...+2`
- 마라의 긴급 소각, 볼트의 생명 담보, 세라의 정보 절취
- gameplay RNG와 분리된 negotiation RNG와 dialogue RNG

일반 UI에는 이름, 감정, Tell, 관계, 조건을 표시하고 숨겨진 목표, 제안 점수, 수락 기준, Tell 신뢰도와 RNG 상태는 DEBUG로 제한했습니다.

## Milestone 5 — Promises, Reputation, and Memory

협상에서 수락한 조건을 현재 또는 다음 라운드까지 유지되는 명시적 약속으로 저장하고, 이행과 배신의 결과가 이후 상호작용에 영향을 주도록 확장했습니다.

구현 범위:

- 이벤트 기반 `PromiseManager`와 런 한정 `PromiseState`
- 경매 불참, 봉인 유지, 카드 보유, 카드 양도, 정보 공유, 상호 패스 약속 6종
- 입찰, 패스, 봉인 해제, 이전, 소각, 정보 제공, 라운드 종료에 따른 자동 판정
- 즉시 보상, 이행 보상과 관계·평판·골드·HP 위약금
- 보상과 위약금의 중복 적용 방지
- 기존 Relationship과 분리된 NPC별 Reputation `-3...+3`
- NPC별 최근 기억 최대 5개와 중요 사건 우선 보존
- 성격, 평판, 관계, 생존 압박, 복수 기억과 숨겨진 목표를 반영한 배신 판단
- gameplay, negotiation, dialogue RNG와 분리된 promise RNG
- 활성 약속, 기한, 위반 조건, 직접 카드 양도·정보 제공과 정산 결과 UI
- 약속 이행·위반·취소 통계와 최종 평판 결과

캐릭터별 배신 경향:

- 마라는 생존 위기 전에는 약속을 비교적 강하게 지킵니다.
- 볼트는 즉시 이익과 유리한 상황을 적극적으로 계산합니다.
- 세라는 정보 가치와 정보 관련 목표를 우선합니다.

평판과 기억은 현재 런에서만 유지되며 새 런에서 초기화됩니다. 내부 약속 ID, 배신 점수, 기억 severity와 RNG 상태는 DEBUG에서만 표시합니다.

- Godot: `4.7.stable.official.5b4e0cb0f`
- 전체 회귀 테스트: `724 assertions`

- 일반 UI의 실제 카드 정보, 숨겨진 목표, 배신 내부 점수 비노출 검증

- 직업, 패시브, 상점과 메타 진행
